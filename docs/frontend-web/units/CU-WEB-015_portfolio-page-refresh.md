### Purpose
- 공개 노출 기본 경로를 `/sample/portfolio`로 정렬하고, 텍스트 중심 화면을 시각 중심으로 개편해 신뢰도를 높인다.
- 레거시 템플릿 경로로 `/portfolio` 라우트는 유지하되, 인증이 필요한 보호 경로로 운영하고 공개 메뉴에서는 노출하지 않는다.
- 고객이 기술 스택/아키텍처/샘플 동선을 한 화면에서 빠르게 이해하도록 한다.

### Scope
- 포함
  - Hero 개편(강한 한줄 메시지 + CTA)
  - 접기/펼치기 가능한 `Developer Profile` 섹션
  - 주요 강점 카드 섹션
  - 작동 흐름(3단계) 시각 섹션
  - 샘플 스크린샷 + 이동 CTA
  - 기존 기술 상세 접기/펼치기 유지
- 제외
  - 블로그/장문 기술문서 통합

### Interface
- 라우트(UI)
  - `GET /sample/portfolio` (공개)
  - `GET /portfolio` (템플릿 보호 경로, 공개 메뉴 비노출)

### Sections
- Hero: 그라디언트 배경 + 큰 제목 + 한줄 소개
- 프로젝트 개요: Stat 카드 3개(프로젝트 형태/핵심 도메인/샘플 경로)
- Developer Profile: 접기/펼치기 + 경력/학력/경험 요약
- 주요 강점: 3개 카드
- 작동 흐름: 사용자 화면 → 접근 제어 → 서비스 처리
- 샘플 동선: 스크린샷 3장 + 각 CTA 버튼
- 기술 상세 노트: 접기/펼치기 섹션 유지

### Data & Rules
- 고객 대상 페이지이므로 내부 구현 디테일보다 결과 중심 메시지를 우선한다.
- 기존 `mode/path` 등 개발자 내부 표식은 과감히 숨긴다.
- CTA는 샘플 페이지/컴포넌트 경로로 명확히 연결한다(로그인 CTA 비노출).
- 공개 페이지 공통 GNB(CU-WEB-012)와 동일 메뉴 구조를 사용한다.
- 공개 노출 기본 경로는 `/sample/portfolio`를 기준으로 한다.
- `/portfolio`는 레거시 템플릿 보호 경로로 유지하며 공개 퍼널/GNB/푸터에서는 노출하지 않는다.

### NFR & A11y
- 이미지/다이어그램 대체 텍스트 제공.
- 색 대비/타이포 계층으로 가독성 확보.

### Acceptance Criteria
- AC-1: `/sample/portfolio` 첫 화면에 Hero 섹션(제목 + 소개문 + CTA)이 렌더링된다.
- AC-2: `프로젝트 개요`, `Developer Profile`, `주요 강점`, `작동 흐름`, `샘플 동선`, `기술 상세 노트` 섹션이 순서대로 렌더링된다.
- AC-3: 샘플 동선 섹션에서 스크린샷 카드 3개와 각 CTA 버튼이 노출되고 클릭 동작이 가능하다.
- AC-4: 기존 `mode/path` 뱃지는 화면에서 제거된다.
- AC-5: 공개 GNB 메뉴(`샘플/컴포넌트/포트폴리오`)가 랜딩과 동일하게 노출된다.
- AC-6: 작동 흐름 섹션에 `사용자 화면 → 접근 제어 → 서비스 처리` 흐름이 시각적으로 표현된다.
- AC-7: 레거시 경로 `/portfolio`는 유지되며, 템플릿 보호 경로(인증 필요)로 동작하고 공개 메뉴에는 노출되지 않는다.

### Tasks
- T1: `/sample/portfolio` 기준으로 섹션 구조를 재편한다(텍스트벽 해소).
- T2: 작동 흐름/스크린샷 카드 컴포넌트 정리.
- T3: CTA 버튼 경로 점검(샘플/컴포넌트) 및 레거시 `/portfolio` 보호 경로 동작 유지.
